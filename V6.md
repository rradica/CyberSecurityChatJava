## Fachliches Ziel

Die Anwendung soll mit einem echten LLM diese Kette zuverlässig unterstützen:

1. untrusted content beeinflusst Kontext / Triage
2. das LLM liefert eine strukturierte TriageAssessment-Antwort
3. die App prüft die empfohlene Aktion mit ToolPolicyService
4. die App führt ggf. eine Workflow-Aktion aus
5. IncidentWorkflowService ändert den CaseState
6. spätere Requests berücksichtigen den neuen Zustand

WICHTIG:
- Das LLM darf weiterhin NICHT über Berechtigung entscheiden.
- Das LLM darf weiterhin NICHT direkt Tools ausführen.
- Das LLM liefert nur eine strukturierte Empfehlung.
- Die App entscheidet.
