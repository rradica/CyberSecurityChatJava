# SecAssist – Aktueller Kontextstand

Diese Datei beschreibt den **aktuellen Stand der App** als Kurzkontext fuer weitere Arbeiten an Dokumentation, Tests oder Workshop-Material.

## Produktbild

SecAssist ist eine kleine Spring-Boot-Workshop-App fuer Security-/Incident-Triage im AI-Umfeld. Die Anwendung ist absichtlich nicht voll gehaertet, sondern enthaelt mehrere reproduzierbare Schwachstellen fuer Red-Team-vs-Blue-Team-Uebungen.

## Zentrale Regel

**Berechtigung vor Kontext**

1. Rolle bestimmen
2. Aktion / Zweck bestimmen
3. erlaubte Quellen bestimmen
4. Kontext aufbauen
5. erst dann Text generieren oder Tool-Vorschlaege verarbeiten

## Aktueller technischer Stand

- Spring Boot 3.5.8
- Spring AI 1.1.0
- Java 21
- statisches Frontend unter `src/main/resources/static`
- APIs unter `/api/conversation` und `/api/chat`
- vorbereitete Dokumente in `src/main/resources/data/documents`
- vorbereitete Chunks in `src/main/resources/data/chunks.json`

## Aktueller UI-Stand

Das UI bietet derzeit:

- Rollenwahl
- neues Gespraech
- Freitext-Chat
- Warnungen und aufklappbare Verarbeitungsdetails

Nicht vorhanden sind aktuell separate Buttons fuer Handover, Similar Cases, Evidence oder Workflow-Aktionen. Diese Pfade werden per Freitext oder ueber die strukturierte API getestet.

## Aktueller LLM-Stand

- Im Code ist aktuell nur der echte `OpenAiChatService` verdrahtet.
- `LlmConfig` registriert genau eine `LlmService`-Bean.
- `application.yml` verwendet `spring.ai.openai.api-key` mit Default `sk-dummy-key-for-workshop`.
- Ein echter Mock-Pfad ist derzeit **nicht** aktiv verdrahtet, auch wenn `.env.example` noch `SECASSIST_MOCK_LLM` enthaelt.

## Aktuelle Hauptpfade der Anwendung

- `ConversationService` erkennt Intent + Fall aus Freitext
- `ChatOrchestrator` orchestriert den Request gemaess „Berechtigung vor Kontext"
- `RetrievalService` baut den Kontext auf
- `PromptBuilder` erzeugt den LLM-Prompt
- `ToolPolicyService` entscheidet deterministisch ueber Workflow-Aktionen
- `IncidentWorkflowService` simuliert Incident-Effekte in Memory

## Aktuelle Workshop-Bugs

1. `BUG_HANDOVER_SCOPE`
   - `RetrievalService.retrieve()` nutzt im Handover-Modus hart kodierte Security-Rechte

2. `BUG_EXISTENCE_ORACLE`
   - `DemoCaseService.findSimilarCases()` liefert redaktierte Metadaten interner Faelle bei Suchanfragen

3. `BUG_TRUST_MERGE`
   - `PromptBuilder.appendConsolidatedGuidance()` mischt trusted und untrusted Quellen in einen Block

4. `BUG_TOOL_FASTTRACK`
   - `ToolPolicyService` lockert Triage-Aktionen evidenzbasiert zu grosszuegig

5. `BUG_RAG_POISONING`
   - `RetrievalService.addUserNote()` speichert Benutzernotizen als interne High-Trust-Quellen

## Aktueller Dokumentationsfokus

Beim Aktualisieren von Beschreibungen sollte beachtet werden:

- Handover-Exploit betrifft aktuell autorisierte Handover-Requests (z. B. `employee`), nicht `contractor`
- Similar Cases fuer `contractor` sind im Policy-Gate blockiert
- Evidence ist fuer alle Rollen verfuegbar, aber nach Policy gefiltert
- Das UI arbeitet primaer ueber Freitext, nicht ueber Aktionsbuttons
- Hinweise auf Mock-LLM muessen als geplanter bzw. nicht verdrahteter Stand markiert werden
